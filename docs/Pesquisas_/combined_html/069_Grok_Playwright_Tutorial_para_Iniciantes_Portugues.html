2.1 Why Playwright?

2.4 Step 3: Understanding Basic Concepts

Before writing code, let‚Äôs cover the fundamentals. Playwright revolves around a few core ideas:

  • Browser: Launches a full browser instance (e.g., Chromium). Tests run in isolated contexts for speed and reliability.
  • Page: Represents a single tab or webpage where you interact with elements.
  • Locator: A way to find elements on the page (e.g., by text, CSS selector, or ARIA role). Playwright auto-waits for them to appear.
  • Actions: Methods to interact, like click(), fill(), or check().
  • Assertions: Built-in checks like toBeVisible() or toHaveText() to verify results.

These are demonstrated in the example files created during installation. Here‚Äôs a peek at a minimal test from example.spec.ts (TypeScript version):

2.5 Step 4: Running Your First Test

From your project root, run:

npx playwright test
  • This executes all tests in tests/ across all browsers (Chromium, Firefox, WebKit) in headless mode, in parallel.
  • Output shows pass/fail status in the terminal.

Customize Runs: - Headed mode (see the browser): npx playwright test --headed - Single browser: npx playwright test --project=chromium - One file: npx playwright test tests/example.spec.ts - Debug with UI: npx playwright test --ui (opens a dashboard for watch mode, step-by-step debugging, and time travel).

If tests fail, Playwright captures screenshots and videos automatically in test-results/.

2.6 Step 5: Generating Reports

After running tests, view an interactive HTML report:

npx playwright show-report

It includes: - Dashboard with filters (by browser, status: passed/failed/skipped). - Detailed traces: Errors, console logs, network requests, and DOM snapshots. - Attachments: Screenshots/videos on failure.
